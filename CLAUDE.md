# CLAUDE.md ‚Äî Pixemingle

## What is this project?

AI-powered pixel art dating platform. Users create pixel avatar agents imbued with their personality. Agents flirt on the user's behalf in animated "theater" sequences generated by Claude. If agents click, users match and chat. Entertainment-first dating ‚Äî comedy is the product, connection is the byproduct.

**Live URL:** Deploying to Vercel (not yet live)
**Launch target:** March 9, 2026

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 14 (App Router, `src/` directory) |
| Language | TypeScript (strict mode) |
| Database / Auth / Realtime | Supabase (Postgres + Auth + Storage + Realtime channels) |
| Styling | Tailwind CSS 3 |
| Animation Engine | Custom HTML5 Canvas (extracted from pixel-agents) |
| LLM | Anthropic Claude API (`@anthropic-ai/sdk`) ‚Äî Haiku for dev, Sonnet for prod |
| Payments | Stripe (subscriptions + microtransaction cosmetics) |
| Hosting | Vercel |
| Tilesets | Modern Interiors by LimeZu (48x48 tiles) |

## Core Concepts

### Journey State Machine
```
HOME_IDLE ‚Üí RESEARCHING ‚Üí BROWSING ‚Üí PROPOSING ‚Üí WAITING ‚Üí THEATER ‚Üí POST_MATCH ‚Üí HOME_IDLE
```
- Managed by `useJourneyState` hook
- Recoverable on page reload via `/api/journey/state`
- Single-page `/world` ‚Äî user never leaves the canvas

### Roles
- **Chaser**: Initiates, picks venue, agent approaches gatekeeper in theater
- **Gatekeeper**: Receives invitations, can accept/counter/decline venue proposals
- Both are persistent user roles chosen during onboarding

### Soul Types
Four personality archetypes that determine agent behavior:
- `romantic` ‚Äî persistence 4, drama 4, romance 5, self-deprecating humor
- `funny` ‚Äî persistence 3, drama 3, romance 2, slapstick humor
- `bold` ‚Äî persistence 2, drama 5, romance 4, dry humor
- `intellectual` ‚Äî persistence 3, drama 2, romance 3, wordplay humor

### Venues (6 date scenes)
`lounge | gallery | japanese | icecream | studio | museum` ‚Äî each has two PNG layers (background + foreground) rendered via canvas.

### Character System
- 20 premade characters (48x48 LimeZu PNGs)
- Custom characters: composite layers (body + eyes + outfit + hairstyle)
- Built via `spritesheetLoader.ts` ‚Üí `buildCharacterSheet()` ‚Üí composited canvas
- `CharacterAppearance` type: `{ body, eyes, outfit, hairstyle, premadeIndex? }`

### Theater (FlirtScenario)
- Claude generates JSON scenario with timed animation steps
- `sequencePlayer.ts` drives playback step-by-step
- Gender-aware animations via `genderAnimations.ts`
- Actions: idle, nervous_walk, pickup_line, blush, victory_dance, etc. (24 total)
- Public replay available at `/theater/[matchId]`

## Database Schema (Supabase/Postgres)

Key tables: `users`, `matches`, `scenarios`, `chat_messages`, `notifications`, `purchases`, `cosmetics`, `reports`, `blocks`, `openclaw_agents`

- `users.soul_type`: romantic | funny | bold | intellectual
- `users.role`: chaser | gatekeeper
- `users.agent_appearance`: JSONB (CharacterAppearance)
- `users.tier`: free | wingman | rizzlord
- `matches.status`: pending_b | active | rejected | expired | unmatched
- `matches.proposed_venue / final_venue`: VenueName
- `scenarios.scenario_data`: JSONB (FlirtScenario)
- Realtime enabled on: `chat_messages`, `notifications`, `matches`

## Key Patterns & Conventions

- **Path alias**: `@/*` maps to `./src/*`
- **Server components** by default, `'use client'` only where needed
- **API routes**: Use `createServerClient` from `@supabase/ssr` for auth
- **Dev auth bypass**: Check `dev-user-id` cookie + `x-dev-user-id` header in API routes
- **LLM calls**: Always in server-side API routes, never client-side. Use `stripCodeFences()` on responses. Retry once on JSON parse failure.
- **Canvas rendering**: All game rendering through the engine ‚Äî no DOM elements inside the canvas area (except photo overlays positioned absolutely)
- **Character sprites**: Async-loaded via `ensureCharacterSheet()`, falls back to `SpriteData` while loading
- **Realtime**: Supabase channels for chat, notifications, match status, venue proposals
- **React Strict Mode**: Be aware of double-effect in development ‚Äî use refs for canvas operations, not closure flags

## Don'ts

- Don't import `@anthropic-ai/sdk` in client components ‚Äî server-only
- Don't use `process.env.SUPABASE_SERVICE_ROLE_KEY` in client code
- Don't modify engine constants without understanding the rendering pipeline
- Don't create new pages ‚Äî the game is a single-page `/world` experience
- Don't commit `.env.local` or any secrets
